מאת שי מרדכי

הרבה מאוד משתמשי אנדרואיד (ואפילו מפתחים) לא מודעים לנתון הבא: "עולם האנדרואיד" כפי שאנחנו מכירים אותו, לא באמת קיים ברגע שהמכשיר נדלק. בשניות הראשונות, הטלפון שלכם פשוט פלטפורמת Linux רגילה לחלוטין.

במאמר זה, ננתח חלק מהנוזקה Anatsa - כלי תקיפה ממשפחת ה-Banking Trojans, המיועד לגניבת פרטי גישה לחשבונות פיננסיים. הנוזקה הזו, מבצעת עקיפה לא שיגרתית של הצגת ה-UI של האפליקציה.

Application vs. Activity - שתי נקודות הכניסה

הפקודה הראשונה של ה-AMS נקראת bindApplication. היא הטריגר שגורם למערכת ההפעלה ליצור את אובייקט ה-Application. הנה ה-Manifest של הנוזקה:

<application
    android:label="PDF Reader: Update"
    android:icon="@vista6/vista_res_0x7f060000"
    android:name="ihp.opjhprs.fsthjpoyr.rhhkkff"
    android:allowBackup="true"
    android:usesCleartextTraffic="true">

התכונה android:name גורמת למערכת ליצור אובייקט Application מותאם אישית. הקוד ב-Entry Point הזה חייב להיות קצר וקטלני:

public class rhhkkff extends Application { 
    @Override
    protected void attachBaseContext(Context base) { 
        super.attachBaseContext(base); 
        new eomqmgsskufu(base).oqwrkywwlykpu(); 
        new tttlkqttg(base).mopksfpffv(); 
    }

הדרך לנצח את המירוץ: Zygote Hook ב-Frida

מכיוון שאי אפשר להתחבר מאוחר (Attach), אנחנו נזריק קוד שיכריח את המערכת ליצור אובייקט Default Application נקי, למרות ה-Manifest הזדוני:

Java.perform(function() {
    var reset_entry_point = Java.use("android.app.LoadedApk");
    
    reset_entry_point.makeApplicationInner.implementation = function(forceDefaultAppClass, instrumentation, allowDuplicateInstances) {
        forceDefaultAppClass = true;
        return this.makeApplicationInner(forceDefaultAppClass, instrumentation, allowDuplicateInstances);
    };
});
