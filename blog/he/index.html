Breaking Trust Boundaries: Core Research Findings

As a Security Researcher, my focus is on identifying logical flaws where execution environments (סביבות ריצה) and trust boundaries meet. Below are the key architectural flaws I have recently uncovered:

  • Android OS & Native Boundaries: Mapped 3 early-stage entry points used by malware to execute code before UI initialization, traced via Frida. Analyzed how JNI acts as an unsafe bridge, stripping protections when crossing to Native layers. Current Research: Hunting a logical flaw to create a UI window during the Application phase, bypassing OS permissions strictly enforced at the Activity phase.
  • AI Agents & Web Isolation: Discovered a logical flaw allowing an Anthropic AI agent to run in the browser without a proper sandbox (ארגז חול), which was patched a month after discovery. Additionally, identified a communication breakdown between a browser extension (תוסף לדפדפן) and a Desktop application.
  • Cloud Architecture & Networking: Mitigated a Backend-to-Frontend logical data leak in a SaaS product by implementing a Rust-based Envoy Tunnel to filter incoming backend variables. Also uncovered a router configuration logical error (TTL=64) that facilitates DNS Rebinding attacks.
  • Compilers & V8 Engine: Identified 3 compile-time logical errors in SBCL's lexical scope (טווח הכרה של משתנים). In the V8 engine, investigated a paradox where standard data inputs unexpectedly transform into executable code (קוד בר-הרצה) during runtime.
